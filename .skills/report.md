# /report - 日报生成指令

## 触发方式
用户说 `/report` 或 "生成日报"

## 执行流程

### 1. 读取今天的工作台
- 打开 `00_工作台/{今天日期}.md`

### 2. 分析勾选状态
扫描 `### 📥 待办` 区域：
- `- [x]` 的条目 → 归入 **已完成**
- `- [ ]` 的条目 → 归入 **进行中**（说明今天碰了但没做完）或 **未开始**（完全没碰）

扫描 `### ⚠️ 遗留/新增问题` 区域 → 归入 **遗留问题**

### 3. 填充日报生成区
将文件底部的 `日报生成区` 更新为：

```markdown
> **日报生成区**
>
> **已完成：**
> 1. xxx
> 2. xxx
>
> **进行中：**
> 1. xxx（说明进度/阻塞原因）
>
> **遗留问题：**
> 1. xxx
```

### 4. 同时输出纯文本版本
在聊天中直接输出一份可复制的纯文本日报，方便用户直接粘贴到 POPO/邮件/易协作：

```
【日报】{今天日期}
✅ 已完成：
1. xxx
2. xxx

🔄 进行中：
1. xxx

⚠️ 遗留问题：
1. xxx
```

## 补充规则
- 子任务跟随父任务归类
- 如果所有任务都完成了，给用户一句鼓励
- 日报语言简洁专业，适合发给团队/领导
