# /context - 相关背景加载指令

## 触发方式
用户说 `/context [关键词]` 或 "加载 [关键词] 相关背景"
例如：`/context 气盾机制`、`/context 战斗系统`

## 执行流程

### 1. 全库搜索相关内容
在以下目录中搜索包含该关键词的所有文件：
- `01_游戏设计/`（设计文档）
- `02_参考资料/`（竞品分析、行业参考）
- `03_会议纪要/`（相关会议决策）
- `05_复盘总结/`（历史复盘）
- `00_工作台/`（近 7 天的工作台，了解近期进展）

### 2. 整理并汇总背景信息
读取搜索到的文件后，按以下结构输出摘要：

```
📚 【{关键词}】背景加载完成

🗂️ 找到相关文档 {N} 篇：
1. {文件名} — {一句话说明核心内容}
2. ...

📌 核心设计现状（当前版本结论）：
{从最新的文档/会议纪要中提炼出现有方案的核心结论}

🔄 主要迭代轨迹：
{按时间顺序列出关键变更节点}

⚠️ 已知问题 / 待确认事项：
{从遗留问题、会议待办中提取}

💡 可供参考的竞品 / 行业资料：
{从 02_参考资料/ 中提取相关内容}
```

### 3. 告知用户背景已加载
输出完成后说明：
- AI 已将上述内容作为本次对话的上下文背景
- 用户现在可以直接提问或开始写新方案，AI 会基于这些背景给出更精准的建议

## 补充规则
- 如果没有找到任何相关文档，告知用户并建议先创建对应的基础文档
- 优先展示最近修改的文档（近期内容权重更高）
- 不要遗漏 `03_会议纪要/` 中的决策结论，这往往是最终定论
