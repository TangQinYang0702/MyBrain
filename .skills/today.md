# /today - 每日工作台生成指令

## 触发方式
用户说 `/today` 或 "生成今日工作台"

## 执行流程

### 1. 定位昨天的工作台
- 在 `00_工作台/` 目录下查找最近一天的日期文件（格式：`YYYY-MM-DD.md`）
- 如果昨天的文件不存在，向前查找最近的工作台文件

### 2. 分析昨天的内容
读取该文件后，识别以下三类内容：
- `- [x]` → **已完成**（归档，不搬运）
- `- [ ]` → **未完成**（搬运到今天）
- `### ⚠️ 遗留/新增问题` 区域下的内容 → **遗留问题**（搬运到今天）
- `### 💬 备注/随手记` → 不搬运，留在原文件

### 3. 生成今天的工作台文件
文件名：`{今天日期}.md`（如 `2026-03-26.md`）
路径：`00_工作台/{YYYY-MM-DD}.md`

模板：
```markdown
# 每日工作台

## {YYYY-MM-DD} {星期X}

### 📥 待办
> 以下从昨日搬运：
{未完成的任务，保持 - [ ] 格式和缩进层级}

> 以下从昨日遗留问题转入：
{遗留问题，转为 - [ ] 格式}

> 新增：
- [ ] 

### ⚠️ 遗留/新增问题


### 💬 备注/随手记


---

> **日报生成区**（下班前说"生成日报"，AI 自动填充）
>
> 已完成：
>
> 进行中：
>
> 遗留问题：
```

### 4. 输出确认
生成后向用户汇报：
- 从昨天搬运了几项未完成任务
- 搬运了几条遗留问题
- 昨天完成了几项（表扬一下）
- 提醒用户在 Obsidian 中打开新文件开始今天的工作

## 补充规则
- 如果今天的文件已存在，询问用户是否覆盖
- 星期使用中文（星期一~星期日）
- 保持任务的原始缩进层级（子任务关系）
- 已完成的子任务如果父任务未完成，子任务标记保持原状搬运
