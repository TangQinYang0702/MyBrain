# 策划的"代码翻译官"：用 AI 快速理解技能编辑器与游戏系统

## 核心思路

用 AI Agent（CodeMaker）**实时读取项目代码**，把 C++/Lua 翻译成策划能看懂的语言。

> 解决痛点：想知道某个节点什么意思 → 等程序回复 → 过两天又忘了

**原理**：VSCode 打开项目代码目录 → CodeMaker 自动获得读取所有代码的能力（与[[第二大脑搭建指南]]的关联方式完全相同）

---

## 环境搭建（10 分钟）

1. 安装 VSCode + CodeMaker 插件
2. VSCode → **文件** → **打开文件夹** → 选择项目代码目录
   - 示例：`X:\L46\L46_workspace_trunk\FortySix`
3. 验证：在 CodeMaker 对话框输入 `帮我看看 Source/.../Ability/Logic/ 目录下有哪些文件`，能列出文件即成功 ✅

> ⚠️ VSCode 打开的文件夹 = AI 能访问的范围，必须打开项目根目录

## 五种核心提问模式

### 📸 模式一：截图提问——"这个是什么？"

**适用**：看到不认识的节点/事件时

```
[贴技能编辑器截图]
这个 CreateBullet 是干嘛用的？
```

AI 流程：识别截图关键词 → 搜索对应类定义 → 阅读代码注释 → 翻译成策划语言

---

### 🔍 模式二：需求反查——"我想要XX效果，有没有？"

**适用**：脑子里有效果，不知道编辑器里有没有现成的

```
我想要一个效果直接生效，不需要打击盒检测，有没有类似的 LogicEvent？
```

---

### 📋 模式三：参数详解——"这个参数到底什么意思？"

**适用**：看到参数名，望文生义不确定真正含义

```
CreateBullet 里的"结算间隔"和"结算次数"具体是怎么配合工作的？
我想让一个持续 3 秒的打击盒每 0.5 秒伤害一次，该怎么配？
```

---

### 🗺️ 模式四：全景扫描——"这个系统有哪些可用功能？"

**适用**：想全面了解某系统有哪些可配置的事件/节点

```
帮我列一下技能编辑器里所有可用的 LogicEvent，按功能分类整理
```

**常见 LogicEvent 分类参考**：

| 分类 | 典型事件 | 用途 |
|------|----------|------|
| 🗡️ 伤害判定 | CreateBullet, DirectEffect, HitCheck | 造成伤害 |
| 🛡️ 防御相关 | CreateDefBox, InvincibilityFrame | 防御/无敌 |
| 🏃 移动控制 | SkillMove, MoveTo, ForbidMove | 技能位移 |
| 🎬 动画表现 | WeaponTrial, CameraShake, NiagaraEffect | 视觉/音效 |
| 🔄 连招逻辑 | ComboInput, ComboSwitch, CanCancel | 连招窗口 |
| 💫 Buff/状态 | AddBuff, AddTag, LastingEffect | 状态效果 |
| 📷 镜头控制 | CameraFov, CameraShake, SwitchCameraMode | 镜头表现 |

---

### 🆚 模式五：对比分析——"A 和 B 有什么区别？"

**适用**：两个相似功能，不确定该用哪个

```
CreateBullet 和 LaunchProjectile 有什么区别？什么情况该用哪个？
LastingEffect 和 DirectEffect 的区别是什么？
```

## 提问四要素

| 要素 | ❌ 错误示范 | ✅ 正确示范 |
|------|-----------|-----------|
| 越具体越好 | "技能系统怎么用？" | "CreateBullet 的穿透参数有哪些？" |
| 带上下文 | "这个参数什么意思？" | "CreateBullet 里 bPenetrateDestructOnly 勾上和不勾有什么区别？" |
| 说明目的 | "有没有移动相关的事件？" | "我想让怪物释放技能时向前突进，有没有现成 LogicEvent？" |
| 善用截图 | 用文字描述界面 | 直接截图贴入对话框 |

## 高频实战案例（直接复用）

```
// 理解未知节点

帮我把 CreateBullet 的所有参数整理成策划可读的配置说明文档

// 跨系统关联查询
从 CreateBullet 触发到最终扣血，中间经过了哪些步骤？
```

## ⚠️ 人机边界

> **AI 翻译代码，策划做决策。AI 提供选项，人来拍板。**

| ✅ 让 AI 做的 | ❌ 不让 AI 做的 |
|-------------|--------------|
| 解释代码功能和参数含义 | 替你决定技能用哪种实现方案 |
| 搜索和推荐可用的 LogicEvent | 替你配置技能编辑器的具体数值 |
| 对比不同功能的区别 | 直接修改项目代码（除非明确要求）|
| 整理参数文档 | 编造代码中不存在的功能 |
| 翻译程序的技术说明 | 替代与程序同学的正式技术评审 |
