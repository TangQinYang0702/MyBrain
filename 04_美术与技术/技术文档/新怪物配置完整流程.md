# 新怪物配置完整流程

> 基于 `DT_MonsterConfig.h` 源码整理，共涉及 **8 张配置表**。
> 
> ⚠️ **必填表**用 🔴 标注，**可选/有则填**用 🟡 标注

---

## 总览：8 张表的关系图

```
🔴 Monster 表（主表）
  ├── Combat 字段 ──────────→ 🔴 Combat 表（战斗数值）
  ├── AIConfig 字段 ─────────→ AI 配置表（另配）
  ├── MonsterEvent 字段 ────→ 🟡 MonsterEvent 表（状态事件）
  ├── MonsterPart 字段 ─────→ 🟡 MonsterPart 表（部位破坏）
  │     └── PartEvent 字段 ─→ 🟡 MonsterPartEvent 表（部位事件）
  ├── MonsterAudioID 字段 ──→ 🟡 MonsterAudio 表（音效）
  ├── AnimRes 字段 ─────────→ 🟡 MonsterAnimDic 表（动画字典）
  └── Blueprint 字段 ───────→ UE蓝图资产路径
```

---

## 🔴 第一步：Monster 表（主表，必填）

**表名**：`Monster`  
**作用**：怪物的基础档案，所有其他表都通过此表的 ID 关联。

| 字段 | 含义 | 是否必填 | 备注 |
|------|------|----------|------|
| ID | 怪物唯一编号 | ✅ 必填 | 全局唯一，其他表引用此 ID |
| MonsterName | 怪物名称 | ✅ 必填 | 显示在 UI 上的名字 |
| MonsterType | 怪物类型 | ✅ 必填 | 影响 AI 行为分类 |
| AttackType | 攻击类型 | ✅ 必填 | 近战/远程等 |
| Blueprint | 蓝图文件路径 | ✅ 必填 | UE 中对应的 BP 路径 |
| Combat | 战斗数值 ID | ✅ 必填 | 关联 Combat 表的 ID |
| AIConfig | AI 配置 ID | ✅ 必填 | 关联 AI 配置表 |
| AIRes | AI 配置 DA 路径 | ✅ 必填 | AI 行为树资源路径 |
| AnimRes | 动画配置路径 | ✅ 必填 | 关联 MonsterAnimDic |
| MonsterDataAssets | 怪物 DA 数据 ID | ✅ 必填 | 动画资源 DataAsset |
| AIPriority | 群体 AI 优先级 | 🟡 选填 | 多怪时的行动优先级 |
| Tags | 怪物标签 | 🟡 选填 | 影响 Buff/技能判定 |
| MonsterEvent | 怪物事件 ID 列表 | 🟡 选填 | 关联 MonsterEvent 表 |
| MonsterPart | 部位 ID 列表 | 🟡 选填 | 有部位破坏时填 |
| MonsterAudioID | 音效 ID | 🟡 选填 | 关联 MonsterAudio 表 |
| AimingTarget | 瞄准锁定骨骼位置 | 🟡 选填 | 锁定时瞄准哪个骨骼 |
| InAimPointAreaRadius | 自动瞄准内框【高宽】| 🟡 选填 | — |
| OutAimPointAreaRadius | 瞄准锁定外框【上下左右】| 🟡 选填 | — |
| HeavyHitCD | 大受击 CD | 🟡 选填 | 连续大受击的间隔限制 |
| SwitchAttachment | 是否允许子弹附着 | 🟡 选填 | — |
| BagTime | 死亡出盒时间 | 🟡 选填 | 死亡后掉落宝箱的延迟 |
| DropBox | 掉落宝箱 ID | 🟡 选填 | — |
| DropBoxScale | 掉落宝箱缩放 | 🟡 选填 | — |
| Ragdoll | 布娃娃表现 ID | 🟡 选填 | 死亡布娃娃效果 |
| canStun | 软鞭武器缠绕功能 | 🟡 选填 | — |
| KillInfo | 击败信息是否显示 | 🟡 选填 | — |
| ColorScheme | 配色方案 ID | 🟡 选填 | — |
| MonsterSize | 怪物体型 | 🟡 选填 | — |
| beHitEffect | 受击特效路径 | 🟡 选填 | — |
| BattleThrowBox | 藏宝怪撒宝掉落组 | 🟡 选填 | 特定怪物类型 |
| DeathDissolveSound | 死亡溶解粒子音效 | 🟡 选填 | — |
| MonsterMaterial | 二级材质路径 | 🟡 选填 | — |

## 🔴 第二步：Combat 表（战斗数值，必填）

**表名**：`Combat`  
**作用**：怪物的所有战斗相关数值，由 Monster 表的 `Combat` 字段引用。

| 字段 | 含义 | 备注 |
|------|------|------|
| ID | 战斗数值唯一 ID | Monster 表里 Combat 字段填这个 |
| HpMax | 最大生命值 | — |
| MonsterATK | 攻击力 | — |
| ArmorLevel | 物理防御力 | — |
| MagicArmorLevel | 魔法防御力 | — |
| WalkSpeed | 移动速度 | — |
| SkillIds | 技能列表 | 怪物可以使用的技能 ID 数组 |
| PassiveSkills | 被动技能列表 | — |
| InitBuffs | 初始 Buff 列表 | 怪物出生自带的 Buff |
| InitTags | 初始 Tag 列表 | 怪物出生自带的 Tag |
| MonsterWeightGear | 受击硬直档位 | 影响被打时的硬直表现 |
| MonsterWeightIdleOffset | 待机状态重量修正 | — |
| BeHitHpMax | 最大韧性值 | 韧性被打空触发大受击 |
| MonTenacityRecover | 韧性恢复速度 | — |
| MonTenacityPauseTime | 韧性暂停恢复时间 | 被打后多久才开始恢复 |
| MonTenacityResetTime | 韧性重置时间 | 长时间没被打后重置 |
| MonImpactLimit | 冲击力下限 | 低于此值的攻击不扣韧性 |
| ResetDelayAfterBreak | 破韧后重置延迟 | 触发大受击后多久重置韧性 |
| ResetRatioOnBreak | 破韧后重置比例 | 重置到满韧性的多少比例 |
| FlyDamageDec | 飞行伤害减免 | — |

---

## 🟡 第三步：MonsterEvent 表（状态事件，选填）

**表名**：`MonsterEvent`  
**作用**：怪物进入/离开某种状态时触发事件（如血量低于 X% 时改变行为）。

| 字段 | 含义 |
|------|------|
| ID | 事件 ID（Monster 表 MonsterEvent 字段引用）|
| Tags | 触发此事件的 Tag 条件 |
| EnterEvents | 进入事件时执行的事件列表 |
| LeaveEvents | 退出事件时执行的事件列表 |

---

## 🟡 第四步：MonsterPart 表（部位破坏，选填）

**表名**：`MonsterPart`  
**作用**：配置怪物的可破坏部位（如打断尾巴、击落翅膀等）。

| 字段 | 含义 |
|------|------|
| MonsterID | 关联的怪物 ID |
| BoneName | 部位对应的骨骼名 |
| Prop | 部位初始属性（持盾值等）|
| PartEvent | 部位事件 ID 列表（关联 MonsterPartEvent）|

---

## 🟡 第五步：MonsterPartEvent 表（部位事件，有部位时填）

**表名**：`MonsterPartEvent`  
**作用**：部位满足条件时触发的事件（如部位血量 < 30% 触发破坏表现）。

| 字段 | 含义 |
|------|------|
| ID | 事件 ID |
| ConditionType | 条件与/或关系 |
| HpPercent | 触发条件：部位生命值百分比 |
| ImpactPercent | 触发条件：持盾值百分比 |
| HasTag | 触发条件：当前持有某 Tag |
| NoHasTag | 触发条件：当前不持有某 Tag |
| MonsterPartEvent | 触发的事件列表 |

---

## 🟡 第六步：MonsterAudio 表（音效，选填）

**表名**：`MonsterAudio`  
**作用**：配置怪物在不同状态下播放的音效。

| 字段 | 含义 |
|------|------|
| ID | 音效组 ID（Monster 表 MonsterAudioID 字段引用）|
| BattleStartAudio | 进战音效 |
| PatrolAudio | 巡逻音效列表 |
| PatrolAudioCD | 巡逻音效触发间隔 |
| SleepAudio | 睡眠/待机音效 |
| ApproachAudio | 追逐音效 |
| HitSAudio | 小受击音效 |
| HitSAudioCD | 小受击音效触发间隔 |
| Ragdoll_BodyImpact | 布娃娃倒地音效 |
| Ragdoll_WeaponImpact | 布娃娃武器掉落音效 |

---

## 🟡 第七步：MonsterAnimDic 表（动画字典，选填）

**表名**：`MonsterAnimDic`  
**作用**：怪物动画资源的键值对映射，AI 系统通过 GameplayTag 查找对应动画。

| 字段 | 含义 |
|------|------|
| Action | 蒙太奇动作字典（Tag → Montage 路径）|
| Base | 基础动画字典（Tag → Animation 路径）|

---

## 🟡 第八步：MonsterUEpath 表（UE 资源路径，选填）

**表名**：`MonsterUEpath`  
**作用**：统一管理怪物相关的 UE 资源路径，便于异步加载。

| 字段 | 含义 |
|------|------|
| ID | 资源 ID |
| AssetPath | 资源的软引用路径 |

---

## ✅ 新怪物配置最小必做清单

```
□ 1. Monster 表：新建一行，填写基础信息、绑定蓝图路径
□ 2. Combat 表：新建战斗数值行，填写 HP/ATK/防御/技能列表
□ 3. 将 Combat 行的 ID 填入 Monster 表的 Combat 字段
□ 4. 确认 AIConfig / AIRes 已关联（AI 行为树）
□ 5. 确认 AnimRes / MonsterDataAssets 已关联（动画资源）
```

**有以下功能时额外配置：**
```
□ 有状态阶段变化 → MonsterEvent 表
□ 有可破坏部位   → MonsterPart 表 + MonsterPartEvent 表
□ 有专属音效     → MonsterAudio 表
□ 有复杂动画字典 → MonsterAnimDic 表
```
